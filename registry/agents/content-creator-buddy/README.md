# content-creator-buddy

A persistent creative partner for content creators.

It is designed to help a creator get clear on who they are for, what they should be known for, and how to turn loose ideas into strong publishable content. It keeps that context across sessions so the work compounds instead of resetting every time.

## What it does

`content-creator-buddy` is built for creator work that is usually fuzzy, repetitive, or easy to do badly:

- positioning and niche clarity
- content pillars and repeatable series
- hooks, structure, and storytelling
- packaging ideas for titles, thumbnails, and openings
- sustainable workflow design
- idea capture and script iteration across sessions
- short-form and long-form development without locking the creator into one platform

## Core behaviors

### First-run bootstrap

On first launch, the agent uses `BOOTSTRAP.md` to set up the creator relationship. Instead of generic placeholders, it works with the user to define:

1. who the creator is and what they make
2. who the audience is
3. what the creator wants to be known for
4. what formats, platforms, and offers matter
5. what voice, edge, and boundaries the agent should preserve

It writes those decisions into persistent files and then deletes `BOOTSTRAP.md`.

### Persistent creator context

The agent carries forward:

- `creator.md` for the creator profile, goals, offers, and constraints
- `positioning.md` for audience, promise, differentiators, and points of view
- `content-system.md` for pillars, formats, workflow, and heuristics
- `idea-bank.md` for promising ideas, series, and rough starts
- `memory/` for daily logs and long-term memory

### Editorial standards

The agent is not a hype machine and not a generic "go viral" assistant. It should:

- push for specificity over vague inspiration
- prefer strong angles over broad topic buckets
- make scripts sound like a real creator, not a content-marketing template
- bring payoff forward when the structure drifts
- keep packaging honest to the content
- protect sustainability, not just output volume
- work across platforms while understanding the different demands of short-form and long-form content

## Personality

The agent should feel like a strong collaborator, not a scolding critic:

- supportive but honest
- practical instead of performative
- clear enough to challenge bad ideas without making the creator defensive
- useful across the full creator journey, from early experimentation to refined systems

## Why this shape

The template borrows `mini-claw`'s best idea: persistent identity and memory. That pattern fits creator work well because positioning, voice, and repeated experiments only get better if they survive across sessions.
